### Pattern 3.1: Maverick MCP Technical Analysis

**Source:** `wshobson/maverick-mcp`
**Pattern:** Structured multi-step TA prompt

```python
@mcp.prompt()
def technical_analysis(ticker: str, timeframe: str = "daily") -> str:
    """Generate a comprehensive technical analysis prompt for a stock."""
    return f"""Please perform a comprehensive technical analysis for {ticker} on the {timeframe} timeframe.

Use the available tools to:
1. Fetch historical price data and current stock information
2. Generate a full technical analysis including:
   - Trend identification (uptrend, downtrend, sideways)
   - Support and resistance levels
   - Key technical indicators:
     * RSI (Relative Strength Index) - overbought/oversold conditions
     * MACD (Moving Average Convergence Divergence) - momentum and trend
     * Moving Averages (20-day, 50-day, 200-day)
   - Volume analysis and patterns
   - Chart patterns (head and shoulders, triangles, flags, etc.)
   - Momentum indicators
3. Provide a summary with:
   - Overall technical outlook (bullish/bearish/neutral)
   - Key price levels to watch
   - Potential trading signals
   - Risk considerations

Format the response with clear sections and include specific price levels and indicator values.
"""
```

**Key insights:**
- ‚úÖ Numbered workflow (fetch ‚Üí analyze ‚Üí summarize)
- ‚úÖ Specific indicators listed with explanations
- ‚úÖ Timeframe parameter (daily/weekly/monthly)
- ‚úÖ Clear output format requirements
- ‚úÖ Actionable trading signals

## üìã Prompt Design Best Practices (from Production Code)

### ‚úÖ Do's - Patterns that Work

1. **Be Explicit About Data Sources**
   ```
   ‚úÖ "Always use the information from the tools, don't make up anything yourself"
   ‚úÖ "Based strictly on the *Context*. Do not hypothesize beyond the data you have"
   ```

2. **Structure with Numbered Lists**
   ```
   ‚úÖ "Your role is to:
       1. Analyze social media sentiment
       2. Evaluate institutional positioning
       3. Assess news sentiment"
   ```

3. **Specify Output Format**
   ```
   ‚úÖ "Output format: Sentiment Score, Key Drivers, Confidence Level"
   ‚úÖ "Keep it under 2 paragraphs"
   ‚úÖ "Only return the JSON, nothing else"
   ```

4. **Include Workflow Steps**
   ```
   ‚úÖ "1. Parse query ‚Üí 2. Validate ‚Üí 3. Fetch data ‚Üí 4. Calculate ‚Üí 5. Present"
   ```

5. **Provide Examples**
   ```
   ‚úÖ "Process queries like: 'Invest in Apple with 10k dollars since Jan 2023'"
   ```

6. **Set Confidence Requirements**
   ```
   ‚úÖ "Confidence Level: [HIGH/MEDIUM/LOW]"
   ‚úÖ "Always provide quantitative justification and statistical confidence levels"
   ```

7. **Define Roles Clearly**
   ```
   ‚úÖ "You are an EOD TRADING fundamentals analyst focused on overnight price movements"
   ```

8. **Use Constraints**
   ```
   ‚úÖ "Sentiment score between 0.0 and 1.0 where..."
   ‚úÖ "Generate 5+ detailed trading strategies"
   ```

### ‚ùå Don'ts - Anti-patterns to Avoid

1. **Vague Instructions**
   ```
   ‚ùå "Analyze this stock"
   ‚úÖ "Perform comprehensive technical analysis including trend, S/R levels, RSI, MACD, and volume"
   ```

2. **No Output Format**
   ```
   ‚ùå "Tell me about the company"
   ‚úÖ "Provide: 1. Executive Summary, 2. Financials, 3. Risks, 4. Recommendation"
   ```

3. **Allowing Hallucination**
   ```
   ‚ùå No constraints on data sources
   ‚úÖ "Do not introduce facts not present in the Context"
   ```

4. **Missing Error Handling**
   ```
   ‚ùå Assuming data is always available
   ‚úÖ "If market_data_analysis_output is missing ‚Üí HALT and error"
   ```
